## Фаза 2: Разработка

Используя документацию из ./docs/analysis/ (НЕ перечитывай референсы заново), создай контейнер.

Контекст: я подключаюсь через браузер телефона, работаю с Claude Code в веб-терминале. Контейнер крутится на Unraid.

### Dockerfile

Базовый образ: node:22-slim

Пакеты: git, curl, sudo, ca-certificates

Пользователь:
- Имя: claude, не root, shell /bin/bash
- Должен быть в sudoers (NOPASSWD для обновлений)
- UID/GID 1000 для совместимости с volume permissions

Установка Claude Code:
- Используй способ установки определённый в фазе 1 как лучший
- Claude Code должен быть доступен в PATH пользователя claude

Переменные окружения: COLORTERM=truecolor, TERM=xterm-256color, LANG=en_US.UTF-8

### Entrypoint скрипт (entrypoint.sh)

Порядок запуска контейнера:
1. Установить пароль пользователя из $CLAUDE_PASSWORD
2. Проверить и обновить Claude Code до последней версии (запускать от пользователя claude)
   - Вывести текущую версию
   - Запустить обновление
   - Если обновилось — вывести новую версию
   - Если уже последняя — вывести "Already latest"
   - Записать лог обновления в /home/claude/.claude/update.log с датой
3. Проверить что claude доступен в PATH и работает (claude --version)
4. Проверить что /project примонтирован и не пустой
5. Запустить веб-панель (Node.js сервер)

Все проверки и обновления должны выводиться в stdout контейнера (видно через docker logs).
Если обновление упало — контейнер всё равно запускается, но пишет WARNING в лог.

### Веб-панель (Node.js приложение)

Создай веб-приложение на Node.js (Express + xterm.js + node-pty), используя паттерны описанные в ./docs/analysis/web-console.md и ./docs/analysis/auth-panel.md:

**Auth:**
- Страница логина — минималистичная, тёмная тема, по образцу из анализа
- Логин/пароль задаётся через переменные CLAUDE_USER и CLAUDE_PASSWORD
- Сессия через cookie, не спрашивает пароль при каждом обновлении страницы
- Порт 7681

**Терминал:**
- xterm.js подключенный к бэкенду через WebSocket
- node-pty на сервере спавнит /home/claude/connect.sh от пользователя claude
- Addons: fit (авто-ресайз под экран), web-links
- Мобильная адаптация: терминал занимает весь экран, нет лишних отступов
- Переподключение при обрыве WebSocket (автоматический реконнект с показом статуса)
- Keepalive ping каждые 30 секунд для мобильного соединения

**UI панели:**
- Тёмная тема, минимализм
- Хедер: название "Claude Code", версия, имя проекта
- Кнопки: "Restart Claude", "Update Claude", "New Session"
- Кнопка "Restart" — убивает текущий процесс и перезапускает connect.sh
- Кнопка "Update" — запускает /home/claude/update.sh в терминале
- Кнопка "New Session" — убивает текущий процесс, запускает claude с чистой сессией
- Статус-бар внизу: статус подключения (connected/reconnecting), время сессии
- Всё адаптивно под мобильный экран

### Скрипты внутри контейнера

/home/claude/connect.sh:
- При открытии терминала автоматически запускается
- Если Claude Code не залогинен — запускает claude login
- Если залогинен — cd /project && claude --dangerously-skip-permissions
- При выходе из claude (через /exit) — показывает меню: [r]estart claude, [u]pdate claude, [s]hell, [q]uit
- Это позволяет не терять терминал после завершения сессии

/home/claude/update.sh:
- Обновляет Claude Code до последней версии
- Выводит текущую версию до и после
- Если обновление произошло — показывает изменения
- Если уже последняя — пишет "Already latest"

### docker-compose.yml

Сервис claude-code:
- build: .
- container_name: claude-code
- ports: "7681:7681"
- volumes:
  - /mnt/user/appdata/code-server/TTS_NEW:/project (рабочие файлы)
  - claude-auth:/home/claude/.claude (персистентная авторизация)
- environment:
  - CLAUDE_USER=claude
  - CLAUDE_PASSWORD=claude
- restart: unless-stopped
- healthcheck: curl -f http://localhost:7681 || exit 1 (интервал 30s)

Named volume: claude-auth

Чтобы обновить Claude — просто docker compose restart claude-code

### Важные детали для мобильного использования
- WebSocket keepalive — сессия не умирает при блокировке экрана
- Если соединение оборвалось — xterm.js автоматически реконнектится
- Claude Code поддерживает resume сессий через --resume / --continue
- Мобильная клавиатура должна корректно работать с терминалом

### Мобильный UX/UI (приоритет — использование с телефона)

**Общее:**
- Mobile-first дизайн — телефон основное устройство, десктоп вторичен
- Viewport meta: width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no
- Запрет bounce/overscroll на iOS (overscroll-behavior: none)
- Safe area insets для телефонов с вырезами/островками (env(safe-area-inset-*))
- PWA manifest + service worker для "Add to Home Screen" — открывается как приложение без адресной строки
- Иконка приложения в стиле Claude (оранжевый на тёмном)
- Тема status bar — тёмная (meta theme-color)

**Терминал на мобильном:**
- Терминал занимает 100% экрана (100dvh — dynamic viewport height, учитывает появляющуюся клавиатуру)
- При появлении клавиатуры терминал уменьшается а не скроллится за экран
- Размер шрифта терминала: 14px по умолчанию, кнопки +/- для изменения, сохранять в localStorage
- Touch-события: долгое нажатие = вставка из буфера, свайп влево = отмена (Esc)
- Дополнительная панель кнопок над клавиатурой (как в Termux): Tab, Ctrl, Esc, стрелки ↑↓←→, Pipe |, /, ~
- Кнопки панели достаточно большие для нажатия пальцем (минимум 44x44px по Apple HIG)
- Вибрация при нажатии на кнопки панели (navigator.vibrate(10))

**Хедер:**
- Компактный на мобильном — одна строка, не больше 44px высотой
- Кнопки — иконки без текста на мобильном, с текстом на десктопе
- Hamburger-меню или slide-up sheet для дополнительных действий (Update, New Session)
- Свайп сверху вниз по хедеру — показать расширенную инфу (версия, аптайм, проект)

**Статус-бар:**
- Компактный, прибит к низу экрана НАД клавиатурой
- Цветовая индикация: зелёный = connected, жёлтый = reconnecting, красный = disconnected
- Анимированный индикатор при reconnecting

**Страница логина:**
- Центрирована вертикально
- Автофокус на поле пароля
- Submit по Enter
- Показать/скрыть пароль глазик
- Запомнить меня checkbox (долгоживущий cookie)
- Поля ввода достаточно крупные (минимум 48px высотой)
- Нет зума при фокусе на input (font-size: 16px минимум на iOS)

**Обработка сети:**
- Offline indicator — если потерял сеть, показать оверлей "No connection" вместо пустого экрана
- При восстановлении сети — автореконнект без действий пользователя
- Показывать счётчик попыток реконнекта
- Если реконнект не удался 5 раз — кнопка "Reconnect manually"
- Буферизация ввода при кратковременном обрыве — не терять напечатанное

**Анимации и feedback:**
- Все кнопки с active state (opacity/scale при нажатии)
- Skeleton loader при загрузке терминала
- Плавное появление терминала (fade-in)
- Toast-уведомления для событий (updated, reconnected, error) — не alert()
